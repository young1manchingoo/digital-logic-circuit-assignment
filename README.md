# 2026 Digital Logic Circuit

## 과제 제출 방법

과제는 **원본 저장소를 Fork한 뒤, 본인 저장소에서 작업하고 Pull Request(PR)를 보내는 방식**으로 제출합니다.

제출 절차는 다음과 같습니다.

1. 원본 저장소를 **Fork**합니다.
2. 본인 저장소에서 과제 파일을 업로드합니다.
3. **Pull Request(PR)** 를 생성합니다.
4. 조교가 확인한 뒤 **merge**합니다.

### 제출 파일 규칙

- 제출 파일 이름은 `학번_이름_과제1` 형식으로 작성합니다.
- 과제물은 `.md`, `.pdf` 등 **형식 제한 없이 제출 가능**합니다.
- 다만, **생성형 AI가 내용을 정확히 인식할 수 있는 상태인지 반드시 본인이 확인한 후 업로드**해야 합니다.
- 손글씨로 문제를 작성하거나 풀이한 경우에는, 해당 문서를 **스캔한 뒤 업로드 전에 생성형 AI에서 문제와 풀이가 정상적으로 인식되는지 반드시 확인**해 주세요.
- 제출 형식 자체는 평가 대상이 아니지만, **AI가 내용을 제대로 인식할 수 있어야 합니다.**
- **AI 인식 여부를 확인하지 않고 제출한 과제물은 점수를 부여하지 않습니다.**

---

## 증빙 파일 제출 안내

- 과제 제출 시, **생성형 AI에서 과제물이 정상적으로 인식되는지 확인한 화면 또는 결과를 증빙 파일로 함께 제출**해야 합니다.
- 증빙 파일에는 **문제 및 풀이 내용이 AI에서 올바르게 읽히는 것이 확인되어야 합니다.**
- 본인 저장소에만 업로드하고 **PR을 생성하지 않으면 제출로 인정되지 않습니다.**
- 과제 번호와 맞지 않는 폴더에 업로드한 경우, 감점 또는 미제출 처리될 수 있습니다.
- 파일명이 식별 불가능한 경우, 제출 확인이 어려울 수 있습니다.

---

## 주의사항

- 학생이 이미 자신의 포크 저장소에서 여러 내용을 수정한 상태라면, **Sync fork** 과정에서 충돌이 발생할 수 있습니다.
- 새로운 과제를 제출하기 전에는, **원본 저장소의 최신 변경사항을 본인 포크 저장소에 반영하기 위해 먼저 Sync fork를 진행**해 주세요.

권장 제출 순서는 다음과 같습니다.

1. 이전 PR 상태를 확인합니다.
2. 포크 저장소를 최신 상태로 갱신합니다. (`Sync fork`)
3. 새 과제 파일을 업로드합니다.
4. PR을 생성합니다.
